# 1. 시스템 개요

## 1.1 시스템 소개

PTLPSM(출하관리 시스템)은 공공조달 물류/출하 업무를 통합 관리하는 시스템입니다. 발주부터 납품완료, 자금 정산까지 전체 프로세스를 지원합니다.

### 주요 특징

| 기능 | 설명 |
|------|------|
| **PDF 자동 추출** | 납품요구서 PDF를 업로드하면 AI/OCR로 정보 자동 추출 |
| **모바일 납품확인** | 현장에서 스마트폰으로 서명, 사진 촬영, GPS 위치 기록 |
| **다중 서명** | 현장소장 + 감리원 서명을 통한 승인 체계 |
| **자금 추적** | 선급금 → 기성금 → 잔금 → OEM 지급 전체 관리 |
| **문서 자동 생성** | 납품확인서, 납품완료계, 선급금 문서 등 자동 생성 |

## 1.2 전체 업무 흐름

### 기본 흐름도

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  발주   │ →  │  출하    │ →  │  운송    │ →  │  납품    │ →  │납품완료  │ →  │자금관리  │
│  등록   │    │  등록    │    │  배차    │    │  확인    │    │   계    │    │  정산   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼              ▼
 납품요구서       출하지시서      운송장          모바일서명      3종PDF        선급/기성
 PDF 추출        발행           발행           사진촬영        생성          잔금/OEM
```

### 데이터 관계도

```
발주(Order) ──── 1:N ────→ 출하(Shipment) ──── 1:1 ────→ 운송(Transport)
    │                          │                              │
    │                          │                              │
    │                          └──────────── 1:1 ─────────────┤
    │                                                         ▼
    │                                                  납품확인(Delivery)
    │                                                         │
    │                                                         ▼
    │                                                  인수증 PDF 생성
    │
    ├──── 1:N ────→ 기성차수(Baseline) ──→ 납품확인서/사진대지 PDF
    │
    ├──── 1:1 ────→ 납품완료계(DeliveryDone) ──→ 3종 PDF
    │
    └──── 1:1 ────→ 자금(Fund)
                      ├── 선급금 (1회)
                      ├── 기성금 (N회) ←── 기성차수와 연결
                      └── 잔금 (1회)
```

### 핵심 관계 설명

| 관계 | 비율 | 의미 |
|------|------|------|
| 발주 → 출하 | **1:N** | 발주 1건에 여러 번 분할 출하 가능 |
| 출하 → 운송 | **1:1** | 출하 1건당 운송장 1건 |
| 운송 → 납품확인 | **1:1** | 운송 완료 시 납품확인 1건 |
| 발주 → 기성차수 | **1:N** | 기성금 분할 청구 가능 |
| 발주 → 자금 | **1:1** | 발주별 자금 관리 |

## 1.3 사용자 역할

| 역할 | 권한 | 주요 업무 |
|------|------|----------|
| **시스템 관리자** (SYSTEM_ADMIN) | 전체 기능 | 사용자 관리, 시스템 설정 |
| **리드파워 담당자** (LEADPOWER_MANAGER) | 대부분 기능 | 발주/출하/납품/자금 관리 |
| **현장 소장** (SITE_MANAGER) | 현장 업무 | 납품확인, 서명 |
| **현장 감리원** (SITE_INSPECTOR) | 감리 업무 | 납품확인 검수, 서명 |
| **운송 담당자** (DRIVER) | 운송 업무 | 배송 상태 업데이트 |

## 1.4 용어 정의

| 용어 | 설명 |
|------|------|
| **납품요구번호** | 조달청에서 발행하는 고유 번호 |
| **본계약** | 최초 체결된 계약 |
| **변경계약** | 기존 계약 내용 변경 |
| **추가계약** | 기존 계약에 추가되는 별도 계약 |
| **기성** | 공사/납품 진행에 따른 대금 청구 단위 |
| **기성 차수** | 기성금 청구 순서 (1차, 2차, 3차...) |
| **선급금** | 계약 체결 후 선지급되는 금액 (통상 70%) |
| **잔금** | 납품완료 후 정산되는 나머지 금액 |
| **OEM 지급** | 협력업체에 지급하는 금액 (기성금의 70%) |

## 1.5 업무 프로세스 상세 흐름

### 1.5.1 발주 → 출하 → 납품확인 사이클

│                    예: 품목A 100개, 품목B 50개                       │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        출하 등록 (1차)                               │
│                   품목A: 100개 중 30개 출하                          │
│                   품목B: 50개 중 20개 출하                           │

                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        운송 배차 (1:1)                               │

                             ▼
┌─────────────────────────────────────────────────────────────────────┐

                             ▼
              ┌──────────────┴──────────────┐
              │     남은 물량이 있는가?      │
              └──────────────┬──────────────┘
                    예 │           │ 아니오
                       ▼           ▼
┌──────────────────────────┐  ┌────────────────────────────────────┐
│     출하 등록 (2차)       │  │       모든 물량 납품확인 완료        │
│  품목A: 남은 70개 중 40개 │  │    → 납품완료계 처리 가능 상태      │

### 1.5.2 기성금 청구 프로세스

**납품확인이 완료된 출하**만 기성금 청구에 포함할 수 있습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    납품확인 완료된 출하 확인                          │
│           (인수증 서명 완료 상태의 출하만 청구 가능)                   │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     기성 차수 생성 (1차)                             │
│          청구할 출하 선택 → 품목별 수량/금액 스냅샷 저장              │
│              예: 1차 출하(30개) + 2차 출하(40개) = 70개              │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        서명 URL 발송                                 │
│              현장소장 + 감리원에게 LMS/SMS 발송                       │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌───────────────────┐                    ┌───────────────────┐
│   현장소장 서명    │ ─────────────────→ │    감리원 서명     │
│  (모바일 접속)     │                    │   (모바일 접속)    │
│  PARTIAL_SIGNED   │                    │ SIGNATURE_COMPLETED│
└───────────────────┘                    └─────────┬─────────┘
                                                   ▼
┌─────────────────────────────────────────────────────────────────────┐
│                 납품확인서 + 사진대지 PDF 자동 생성                   │
│                       기성 차수 상태: CONFIRMED                      │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        기성금 수금 확인                              │
│                  입금일, 입금금액 확인 후 PAID 처리                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.5.3 선급금 vs 기성금 vs 잔금

세 가지 자금은 **신청 시점과 조건이 다릅니다**.

| 구분 | 신청 시점 | 신청 조건 | 금액 기준 | 서명 필요 |
|------|----------|----------|----------|----------|
| **선급금** | 첫 출하 전후 (계약 초기) | 출하 여부 무관 | 계약금액 × 비율(70%) | X |
| **기성금** | 납품 진행 중 (여러 차수) | 납품확인 완료된 출하 | 실제 납품수량 × 단가 | O (현장소장+감리원) |
| **잔금** | 납품완료 후 | 납품완료계 완료 | 계약금액 - 선급금 - 기성금누계 | X |

**자금 흐름 예시:**
```
계약금액: 1억원
    │
    ├─→ 선급금: 7천만원 (70%) ←── 첫 출하 전 신청
    │
    ├─→ 1차 기성금: 1천만원 ←── 1차~2차 출하 납품확인 완료 후
    │
    ├─→ 2차 기성금: 1천만원 ←── 3차~4차 출하 납품확인 완료 후
    │
    └─→ 잔금: 1천만원 ←── 납품완료계 서명 완료 후
         (1억 - 7천만 - 1천만 - 1천만 = 1천만)
```

### 1.5.4 납품완료 시 생성 서류

모든 출하의 납품확인이 완료되고, **현장소장 + 감리원 서명**이 완료되면 다음 서류가 자동 생성됩니다:

| 서류 | 내용 | 용도 | 생성 시점 |
|------|------|------|----------|
| **인수증** | 출하별 납품 내역 + 인수자 서명 | 개별 납품 증빙 | 납품확인 완료 시 |
| **납품확인서** | 기성 차수별 품목 내역 + 양측 서명 | 기성금 청구 증빙 | 기성 차수 서명 완료 시 |
| **사진대지** | 현장 사진 모음 (출하당 2장) | 납품 현장 증빙 | 기성 차수 서명 완료 시 |
| **납품완료계** | 전체 납품 요약 보고서 | 조달청 제출 | 최종 서명 완료 시 |
| **납품내역서** | 차수별 청구 내역 정리 | 정산 자료 | 필요 시 다운로드 |

### 1.5.5 상태별 가능 작업

각 단계별로 **가능한 작업이 제한**됩니다:

| 단계 | 상태 | 가능한 작업 | 불가능한 작업 |
|------|------|------------|--------------|
| **출하** | 대기 | 수정, 삭제, 운송 배차 | - |
| **출하** | 진행중 | 추가변경(수량조정) | 삭제 |
| **출하** | 완료 | 조회만 | 수정, 삭제, 추가변경 |
| **기성차수** | 작성중 | 출하 선택 변경 | - |
| **기성차수** | 서명대기 | 서명 URL 재발송 | 출하 변경 |
| **기성차수** | 확정 | PDF 다운로드, 수금 확인 | 모든 수정 |
| **선급금** | 신청 | 취소 | 재신청 |
| **선급금** | 승인/수금완료 | 조회만 | 모든 수정 |

> **중요**: 기성금 청구에 포함된 출하는 수량 변경이 불가능합니다. 수량 조정이 필요한 경우 새로운 출하를 등록해야 합니다.

# 2. 발주 관리

## 2.1 발주 목록 조회

### 접근 경로
**메뉴**: 발주관리 → 발주목록

### 화면 구성

발주 목록은 **트리 구조**로 표시됩니다:
- **본계약**: 최상위 항목
- **변경계약/추가계약**: 본계약 하위에 표시

### 검색 조건

| 검색 항목 | 설명 |
|-----------|------|
| 납품요구번호 | 조달청 발행 번호로 검색 |
| 현장명 | 현장/사업명으로 검색 |
| 수요기관 | 발주처 기관명으로 검색 |
| 계약 유형 | 본계약/변경계약/추가계약 |
| 상태 | 진행중/완료/취소 |
| 등록 기간 | 발주 등록일 범위 |

### 주요 기능

| 버튼 | 기능 |
|------|------|
| **신규 등록** | 새 발주 등록 화면으로 이동 |
| **엑셀 다운로드** | 조회 결과를 Excel로 내보내기 |
| **상세보기** | 행 클릭 시 상세 화면으로 이동 |

## 2.2 발주 등록

### 접근 경로
**메뉴**: 발주관리 → 발주목록 → [신규 등록] 버튼

### 등록 방법

#### 방법 1: PDF 자동 추출 (권장)

1. **PDF 업로드 영역**에 납품요구서 PDF 파일을 드래그하거나 클릭하여 선택
2. 시스템이 AI/OCR로 다음 정보를 자동 추출:
   - 납품요구번호
   - 수요기관명
   - 현장명/사업명
   - 납품 품목 목록
   - 수량 및 금액
3. 추출된 정보 확인 및 수정
4. **[저장]** 버튼 클릭

#### 방법 2: 수동 입력

1. 기본 정보 입력:
   - 납품요구번호 (필수)
   - 계약 유형 선택 (본계약/변경계약/추가계약)
   - 수요기관명
   - 현장명
   - 계약금액
2. 품목 정보 입력:
   - **[품목 추가]** 버튼으로 행 추가
   - 품목명, 규격, 수량, 단가 입력
3. **[저장]** 버튼 클릭

### 입력 필드 설명

| 필드 | 필수 | 설명 |
|------|:----:|------|
| 납품요구번호 | O | 조달청 발행 번호 |
| 계약 유형 | O | 본계약/변경계약/추가계약 |
| 상위 계약 | △ | 변경/추가계약 시 연결할 본계약 선택 |
| 수요기관명 | O | 발주처 기관명 |
| 현장명 | O | 납품 현장/사업명 |
| 계약금액 | O | 총 계약금액 |
| 선급금 비율 | - | 기본값 70%, 필요 시 수정 |
| 납품기한 | - | 최종 납품 완료 기한 |
| 비고 | - | 추가 메모 사항 |

### 주의사항

> **중요**: 변경계약/추가계약 등록 시 반드시 **상위 계약**을 선택해야 합니다. 선택하지 않으면 별도의 본계약으로 등록됩니다.

## 2.3 발주 수정

### 접근 경로
발주 목록 → 해당 발주 클릭 → 상세 화면 → **[수정]** 버튼

### 수정 가능 항목

| 항목 | 수정 가능 조건 |
|------|----------------|
| 기본 정보 | 출하 전까지 수정 가능 |
| 품목 정보 | 해당 품목 출하 전까지 수정 가능 |
| 수량 | 기출하 수량보다 큰 값으로만 수정 가능 |
| 금액 | 기청구 금액에 영향 없는 범위에서 수정 가능 |

### 수정 불가 상황

- 이미 출하가 진행된 품목의 삭제
- 기성금 청구 완료된 건의 금액 변경
- 납품완료 처리된 발주의 핵심 정보

# 3. 출하 관리

## 3.1 출하 목록 조회

### 접근 경로
**메뉴**: 출하관리 → 출하목록

### 화면 구성

출하 목록에서는 각 출하 건의 상태를 한눈에 확인할 수 있습니다.

### 출하 상태

| 상태 | 설명 | 표시 색상 |
|------|------|-----------|
| **등록** | 출하 정보 등록 완료 | 회색 |
| **운송중** | 배송 출발 | 파란색 |
| **배송완료** | 현장 도착 | 주황색 |
| **납품확인** | 납품확인 완료 | 녹색 |

### 검색 조건

| 검색 항목 | 설명 |
|-----------|------|
| 납품요구번호 | 연결된 발주의 납품요구번호 |
| 현장명 | 납품 현장명 |
| 출하일 | 출하 예정일/실제 출하일 |
| 상태 | 등록/운송중/배송완료/납품확인 |
| 운송사 | 배송 담당 운송사 |

## 3.2 출하 등록

### 접근 경로
**메뉴**: 출하관리 → 출하목록 → **[신규 등록]** 버튼

### 등록 절차

#### Step 1: 발주 선택
1. **발주 검색** 영역에서 납품요구번호 또는 현장명으로 검색
2. 출하할 발주 건 선택
3. 해당 발주의 품목 목록이 표시됨

#### Step 2: 출하 품목 및 수량 입력
1. 출하할 품목 선택 (체크박스)
2. 각 품목별 출하 수량 입력
3. 시스템이 자동으로 검증:
   - 출하수량 ≤ 발주수량 - 기출하수량

#### Step 3: 운송 정보 입력
1. 출하 예정일 선택
2. 운송사 선택 (선택사항)
3. 차량번호 입력 (선택사항)
4. 운전자 정보 입력 (선택사항)

#### Step 4: 저장
1. 입력 정보 확인
2. **[저장]** 버튼 클릭
3. 출하 등록 완료

### 입력 필드 설명

| 필드 | 필수 | 설명 |
|------|:----:|------|
| 발주 선택 | O | 출하할 발주 건 |
| 출하 품목 | O | 출하할 품목 (복수 선택 가능) |
| 출하 수량 | O | 각 품목별 출하 수량 |
| 출하 예정일 | O | 출하 예정 날짜 |
| 운송사 | - | 배송 담당 운송사 |
| 차량번호 | - | 배송 차량 번호 |
| 운전자명 | - | 운전자 이름 |
| 연락처 | - | 운전자 연락처 |
| 비고 | - | 출하 관련 메모 |

## 3.3 출하 수정 및 추가변경

### 일반 수정

출하 등록 후 **납품확인 전**까지 다음 항목 수정 가능:
- 출하 예정일
- 운송 정보 (운송사, 차량, 운전자)
- 비고

### 추가변경 기능

이미 등록된 출하의 **수량을 변경**해야 할 경우 사용합니다.

#### 추가변경 절차

1. 출하 상세 화면에서 **[추가변경]** 버튼 클릭
2. 변경 유형 선택:
   - **수량 증가**: 추가 출하 필요 시
   - **수량 감소**: 출하 수량 축소 시
3. 변경 수량 입력
4. 변경 사유 입력 (필수)
5. **[변경 적용]** 버튼 클릭

#### 추가변경 제한

> **주의**: 해당 출하 건이 **기성금 청구에 포함된 경우** 수량 변경이 불가능합니다. 이 경우 새로운 출하를 등록해야 합니다.

### 변경 이력 조회

출하 상세 화면 하단의 **[변경 이력]** 탭에서 모든 수량 변경 기록을 확인할 수 있습니다:
- 변경 일시
- 변경 전/후 수량
- 변경 사유
- 변경자

# 4. 납품 확인

## 4.1 납품확인 개요

납품확인은 현장에서 실제 물품을 수령하고 확인하는 과정입니다. **모바일 기기**를 통해 현장에서 직접 처리할 수 있습니다.

### 납품확인 프로세스

```
출하 완료 (배송완료 상태)
         ↓
   모바일 URL 발송
         ↓
   현장 담당자 접속
         ↓
   수량 확인 및 사진 촬영
         ↓
   터치 서명
         ↓
   납품확인 완료
```

## 4.2 관리자 화면 (PC)

### 접근 경로
**메뉴**: 납품관리 → 납품확인

### 화면 구성

납품확인 목록은 **트리 구조**로 표시됩니다:
- 발주 단위로 그룹화
- 각 발주 하위에 출하 건 목록 표시
- 출하별 납품확인 상태 표시

### 납품확인 URL 발송

1. 납품확인이 필요한 출하 건 선택
2. **[납품확인 URL 발송]** 버튼 클릭
3. 수신자 정보 입력:
   - 수신자명
   - 휴대폰 번호
4. 메시지 유형 선택 (SMS/LMS)
5. **[발송]** 버튼 클릭

### URL 유효기간

- 발송된 URL은 **24시간** 동안 유효합니다
- 만료 시 재발송이 필요합니다

## 4.3 모바일 납품확인

### 접속 방법

1. 문자로 수신한 URL 클릭
2. 모바일 브라우저에서 납품확인 페이지 열림
3. 별도 로그인 불필요 (토큰 기반 인증)

### 납품확인 절차

#### Step 1: 정보 확인
- 납품요구번호
- 현장명
- 납품 품목 및 수량 확인

#### Step 2: 수량 확인
- 각 품목별 실제 수령 수량 확인
- 수량 불일치 시 실제 수량 입력

#### Step 3: 사진 촬영
1. **[사진 추가]** 버튼 터치
2. 카메라 앱 실행
3. 납품 현장/물품 촬영 (최대 5장)
4. GPS 위치 정보 자동 기록

#### Step 4: 서명
1. 서명 영역에 손가락으로 서명
2. 서명이 마음에 들지 않으면 **[다시 서명]** 터치
3. 서명 완료 후 **[확인]** 터치

#### Step 5: 납품확인 완료
1. 모든 정보 최종 확인
2. **[납품확인 완료]** 버튼 터치
3. 확인 완료 메시지 표시

### 오프라인 지원

> **참고**: 네트워크가 불안정한 현장에서도 사용할 수 있습니다. 오프라인 상태에서 작성한 내용은 로컬에 저장되며, 네트워크 연결 시 자동으로 서버에 전송됩니다.

# 5. 납품완료계

## 5.1 납품완료계 개요

납품완료계는 모든 납품이 완료된 후 **조달청에 제출하는 공식 문서**입니다. 현장소장과 감리원의 서명을 받아 완성됩니다.

### 납품완료계 상태 흐름

```
PENDING (대기)
    ↓ 첫 출하 생성
IN_PROGRESS (납품중)
    ↓ 모든 납품확인 완료
PENDING_SIGNATURE (서명 대기)
    ↓ 현장소장 서명
PARTIAL_SIGNED (일부 서명)
    ↓ 감리원 서명
COMPLETED (완료) → 3종 PDF 자동 생성
    ↓ 조달청 제출
SUBMITTED (제출완료)
```

## 5.2 납품완료계 목록

### 접근 경로
**메뉴**: 납품관리 → 납품완료계

### 화면 구성

| 컬럼 | 설명 |
|------|------|
| 납품요구번호 | 조달청 발행 번호 |
| 현장명 | 납품 현장/사업명 |
| 상태 | 현재 진행 상태 |
| 납품확인 현황 | 완료/전체 (예: 5/5) |
| 서명 현황 | 현장소장/감리원 서명 상태 |
| 등록일 | 납품완료계 생성일 |

## 5.3 서명 수집

### 서명 URL 발송

1. 납품완료계 상세 화면 진입
2. **[서명 URL 발송]** 버튼 클릭
3. 서명자 선택:
   - **현장소장**: 담당자 목록에서 선택
   - **감리원**: 담당자 목록에서 선택
4. 메시지 미리보기 확인
5. **[발송]** 버튼 클릭

### 서명 메시지 내용

발송되는 메시지에는 다음 정보가 포함됩니다:
- 납품요구번호
- 수요기관
- 사업명
- 서명 URL (유효기간: 24시간)

### 서명 진행 상태 확인

| 상태 | 아이콘 | 설명 |
|------|--------|------|
| 대기 | ⏳ | 서명 URL 미발송 또는 발송 후 미서명 |
| 완료 | ✅ | 서명 완료 |

## 5.4 PDF 문서 생성 및 다운로드

### 자동 생성 조건

**현장소장 + 감리원** 양측 서명이 모두 완료되면 다음 3종의 PDF가 자동 생성됩니다:

| 문서 | 설명 |
|------|------|
| **납품확인서** | 품목별 납품 내역 확인 문서 |
| **납품완료계** | 최종 납품완료 보고서 |
| **사진대지** | 현장 사진 모음 문서 |

### 다운로드 방법

1. 납품완료계 상세 화면
2. **문서** 탭 선택
3. 원하는 문서의 **[다운로드]** 버튼 클릭
4. 또는 **[전체 다운로드]**로 ZIP 파일 다운로드

## 5.5 조달청 제출

### 제출 처리

1. 모든 서명 완료 확인
2. PDF 문서 생성 확인
3. **[제출 완료 처리]** 버튼 클릭
4. 상태가 **SUBMITTED (제출완료)**로 변경

> **주의**: 제출 완료 처리 후에는 수정이 불가능합니다.

# 6. 자금 관리 - 선급금

## 6.1 선급금 개요

선급금은 계약 체결 후 납품 전에 **선지급**되는 금액입니다. 통상 계약금액의 **70%**를 선급금으로 지급받습니다.

### 선급금 프로세스

```
선급금 신청
    ↓
5종 PDF 문서 자동 생성
    ↓
관리자 승인
    ↓
실제 입금
    ↓
수금 확인 처리
```

## 6.2 자금 상세 화면 접근

### 접근 경로
**메뉴**: 자금관리 → 자금목록 → 해당 건 클릭

### 화면 구성

자금 상세 화면은 **4개의 탭**으로 구성됩니다:
1. **선급금** 탭
2. **기성금** 탭
3. **잔금** 탭
4. **OEM 지급** 탭

## 6.3 선급금 신청

### 신청 절차

1. 자금 상세 화면 → **선급금** 탭
2. **[선급금 신청]** 버튼 클릭
3. 선급금 신청 모달 창 열림

### 입력 정보

| 필드 | 설명 |
|------|------|
| 계약금액 | 자동 표시 (수정 불가) |
| 선급금 비율 | 기본 70%, 필요 시 수정 |
| 예상 선급금액 | 자동 계산 (계약금액 × 비율) |
| 신청 금액 | 실제 신청할 금액 입력 |
| 신청일 | 신청 날짜 선택 |
| 비고 | 추가 메모 (선택) |

### 신청 완료

1. 정보 입력 후 **[신청]** 버튼 클릭
2. 시스템이 **5종 PDF 문서** 자동 생성:
   - 선급금신청서
   - 선급금사용계획
   - 선급금사용확약서
   - 선급금사용각서
   - 선급금정산서
3. 상태: **REQUESTED (신청)**

> **주의**: 선급금은 **1회만 신청** 가능합니다. 신청 후에는 취소할 수 없으니 신중하게 진행하세요.

## 6.4 선급금 문서 다운로드

### 개별 다운로드

1. 선급금 탭의 **문서 목록** 영역
2. 원하는 문서의 **[다운로드]** 아이콘 클릭

### 전체 다운로드

**[전체 다운로드]** 버튼 클릭 → 5종 문서 ZIP 파일 다운로드

## 6.5 선급금 승인

### 승인 권한

시스템 관리자 또는 승인 권한이 있는 사용자만 승인 가능

### 승인 절차

1. 선급금 탭에서 신청 내역 확인
2. **[승인]** 버튼 클릭
3. 승인 금액 확인 (신청 금액과 다를 수 있음)
4. **[승인 확인]** 클릭
5. 상태: **APPROVED (승인)**

## 6.6 선급금 수금 확인

### 수금 확인 절차

실제로 선급금이 입금되면 수금 확인 처리를 합니다.

1. 선급금 탭에서 승인된 내역 확인
2. **[수금 확인]** 버튼 클릭
3. 수금 확인 모달 창에서 정보 입력:

| 필드 | 설명 |
|------|------|
| 입금일 | 실제 입금 날짜 |
| 입금 금액 | 실제 입금된 금액 |
| 입금 계좌 | 입금된 계좌 정보 (선택) |
| 비고 | 추가 메모 (선택) |

4. **[확인]** 버튼 클릭
5. 상태: **PAID (수금완료)**

# 7. 자금 관리 - 기성금

## 7.1 기성금 개요

기성금은 납품이 진행됨에 따라 **단계별로 청구**하는 금액입니다. 납품확인이 완료된 출하 건을 선택하여 기성금을 청구합니다.

### 기성금 프로세스

```
납품확인 완료된 출하 선택
         ↓
   기성 청구 생성
         ↓
   서명 URL 발송 (현장소장 + 감리원)
         ↓
   양측 서명 완료
         ↓
   PDF 자동 생성
         ↓
   상태: REQUESTED
         ↓
   실제 입금 후 수금 확인
         ↓
   상태: PAID
```

## 7.2 기성금 청구

### 청구 조건

- **납품확인이 완료된 출하**만 청구 가능
- 이미 청구에 포함된 출하는 선택 불가

### 청구 절차

#### Step 1: 기성 청구 시작

1. 자금 상세 화면 → **기성금** 탭
2. **[기성 청구]** 버튼 클릭

#### Step 2: 출하 선택

1. 청구 가능한 출하 목록 표시
2. 청구할 출하 건 선택 (복수 선택 가능)
3. 선택한 출하의 합계 금액 자동 계산
4. OEM 지급 예정액 자동 계산 (청구금액 × 70%)

#### Step 3: 서명 담당자 선택

**[다음]** 버튼 클릭 후 서명 담당자 선택:

| 역할 | 설명 |
|------|------|
| 현장소장 | 드롭다운에서 선택 |
| 감리원 | 드롭다운에서 선택 |

#### Step 4: 메시지 미리보기 및 발송

1. 각 담당자에게 발송될 메시지 미리보기
2. 메시지 내용 확인:
   - 납품요구번호
   - 수요기관
   - 사업명
   - 청구금액
   - 서명 URL
3. **[발송]** 버튼 클릭

## 7.3 서명 진행

### 서명 순서

1. **현장소장 서명**: 먼저 서명
2. **감리원 서명**: 현장소장 서명 후 진행

### 서명 상태

| 상태 | 설명 |
|------|------|
| PENDING_SIGNATURE | 서명 대기 중 |
| PARTIAL_SIGNED | 현장소장만 서명 완료 |
| SIGNATURE_COMPLETED | 양측 모두 서명 완료 |

### 서명 완료 후

양측 서명이 완료되면:
1. 기성청구 관련 PDF 자동 생성
2. 상태가 **REQUESTED**로 변경
3. 수금 확인 버튼 활성화

## 7.4 기성금 수금 확인

### 수금 확인 절차

1. 기성금 탭에서 청구 내역 확인
2. 해당 건의 **[수금 확인]** 버튼 클릭
3. 수금 확인 모달에서 정보 입력:

| 필드 | 설명 |
|------|------|
| 입금일 | 실제 입금 날짜 |
| 입금 금액 | 실제 입금된 금액 |
| 입금 계좌 | 입금된 계좌 정보 |
| 비고 | 추가 메모 |

4. **[확인]** 버튼 클릭
5. 상태: **PAID (수금완료)**

## 7.5 기성금 이력 조회

기성금 탭 하단에서 모든 기성 청구 이력을 확인할 수 있습니다:

| 항목 | 설명 |
|------|------|
| 차수 | 1차, 2차, 3차... |
| 청구일 | 기성 청구 날짜 |
| 청구금액 | 해당 차수 청구 금액 |
| 수금일 | 실제 입금 날짜 |
| 수금금액 | 실제 입금 금액 |
| 상태 | 청구/승인/수금완료 |

# 8. 자금 관리 - 잔금 및 OEM

## 8.1 잔금 개요

잔금은 **납품완료 후** 정산되는 나머지 금액입니다.

### 잔금 계산

```
잔금 = 계약총액 - 선급금 - 기성금누계
```

### 계산 기준

| 기준 | 설명 |
|------|------|
| **신청 기준** | 신청된 금액 기준으로 계산 |
| **입금 기준** | 실제 입금된 금액 기준으로 계산 |

## 8.2 잔금 신청

### 신청 조건

- **납품완료 처리** 후에만 신청 가능
- 모든 출하의 납품확인이 완료되어야 함

### 신청 절차

1. 자금 상세 화면 → **잔금** 탭
2. **[잔금 신청]** 버튼 클릭
3. 잔금 신청 모달에서:
   - 계산 기준 선택 (신청 기준 / 입금 기준)
   - 계산 내역 확인:
     - 계약총액
     - (-) 선급금
     - (-) 기성금누계
     - (=) 잔금
   - 신청일 입력
   - 비고 입력 (선택)
4. **[신청]** 버튼 클릭

### 주의사항

> **경고**: 신청 금액과 계산된 잔금이 다를 경우 경고 메시지가 표시됩니다. 금액을 다시 확인하세요.

## 8.3 잔금 수금 확인

### 수금 확인 절차

1. 잔금 탭에서 신청 내역 확인
2. **[수금 확인]** 버튼 클릭
3. 수금 정보 입력:
   - 입금일
   - 입금 금액
   - 입금 계좌 (선택)
   - 비고 (선택)
4. **[확인]** 버튼 클릭
5. 상태: **PAID (수금완료)**

## 8.4 OEM 지급 개요

OEM 지급은 **협력업체에 지급**하는 금액입니다. 일반적으로 기성금의 **70%**를 지급합니다.

### OEM 지급 프로세스

```
기성금 청구 완료
      ↓
OEM 지급 등록 (지급 예정)
      ↓
실제 지급
      ↓
OEM 지급 완료 처리
```

## 8.5 OEM 지급 등록

### 등록 절차

1. 자금 상세 화면 → **OEM 지급** 탭
2. **[OEM 지급 등록]** 버튼 클릭
3. 지급 정보 입력:

| 필드 | 설명 |
|------|------|
| 연결 기성금 | 관련된 기성금 청구 선택 (선택사항) |
| 지급 예정액 | 지급 예정 금액 (기성금 × 70% 자동 계산) |
| 지급 예정일 | 지급 예정 날짜 |
| OEM 업체명 | 협력업체 이름 |
| 계좌 정보 | 지급 계좌 정보 (선택) |
| 비고 | 추가 메모 (선택) |

4. **[등록]** 버튼 클릭
5. 상태: **SCHEDULED (예정)**

### 자동 금액 입력

**[70% 자동 입력]** 버튼을 클릭하면 연결된 기성금의 70%가 자동 입력됩니다.

## 8.6 OEM 지급 완료

### 완료 처리 절차

실제로 OEM 업체에 지급이 완료되면:

1. OEM 지급 목록에서 해당 건 선택
2. **[지급 완료]** 버튼 클릭
3. 지급 완료 정보 입력:

| 필드 | 설명 |
|------|------|
| 지급일 | 실제 지급 날짜 |
| 지급 금액 | 실제 지급 금액 |
| 계좌 정보 | 지급한 계좌 정보 |
| 비고 | 추가 메모 |

4. **[완료]** 버튼 클릭
5. 상태: **PAID (지급완료)**

## 8.7 자금 현황 요약

자금 상세 화면 상단에서 전체 자금 현황을 한눈에 확인할 수 있습니다:

| 항목 | 설명 |
|------|------|
| 계약총액 | 총 계약 금액 |
| 선급금 | 신청/입금 금액 |
| 기성금 누계 | 전체 기성금 합계 |
| 잔금 | 남은 정산 금액 |
| OEM 지급 | 협력업체 지급 총액 |
| **수금률** | (선급금+기성금+잔금) ÷ 계약총액 × 100% |

# 9. FAQ 및 문제해결

## 9.1 자주 묻는 질문

### 발주 관련

**Q: PDF 업로드 후 정보가 정확하게 추출되지 않습니다.**
> A: AI/OCR 추출은 PDF 품질에 따라 정확도가 달라질 수 있습니다. 추출 후 반드시 정보를 확인하고 필요 시 수동으로 수정하세요.

**Q: 변경계약을 등록했는데 본계약과 연결되지 않습니다.**
> A: 변경계약 등록 시 **상위 계약** 필드에서 반드시 본계약을 선택해야 합니다. 선택하지 않으면 별도의 본계약으로 등록됩니다.

### 출하 관련

**Q: 출하 수량을 변경하고 싶은데 변경 버튼이 비활성화되어 있습니다.**
> A: 해당 출하가 이미 **기성금 청구에 포함**된 경우 수량 변경이 불가능합니다. 이 경우 새로운 출하를 등록하거나 기성금 청구를 취소해야 합니다.

**Q: 출하를 삭제하고 싶습니다.**
> A: 납품확인이 시작되지 않은 출하만 삭제할 수 있습니다. 납품확인이 진행된 경우에는 삭제가 불가능합니다.

### 납품확인 관련

**Q: 모바일에서 납품확인 URL에 접속이 안 됩니다.**
> A: URL 유효기간(24시간)이 지났을 수 있습니다. 관리자에게 요청하여 새로운 URL을 발송받으세요.

**Q: 사진을 촬영했는데 업로드가 안 됩니다.**
> A: 네트워크 상태를 확인하세요. 오프라인 상태에서는 로컬에 저장되며, 네트워크 연결 시 자동 업로드됩니다.

**Q: 서명 후 수정이 필요합니다.**
> A: 서명 완료 후에는 수정이 불가능합니다. 관리자에게 문의하여 서명 취소 후 재진행해야 합니다.

### 자금 관련

**Q: 선급금을 다시 신청하고 싶습니다.**
> A: 선급금은 **1회만 신청 가능**합니다. 금액 변경이 필요한 경우 관리자에게 문의하세요.

**Q: 기성금 청구 시 일부 출하가 목록에 표시되지 않습니다.**
> A: 다음 조건을 확인하세요:
> 1. 해당 출하의 납품확인이 완료되었는지
> 2. 이미 다른 기성 청구에 포함되었는지

**Q: OEM 지급 금액을 수정하고 싶습니다.**
> A: **SCHEDULED (예정)** 상태에서만 수정 가능합니다. **PAID (지급완료)** 상태에서는 수정이 불가능합니다.

## 9.2 오류 메시지 및 대응

| 오류 메시지 | 원인 | 해결 방법 |
|-------------|------|-----------|
| "세션이 만료되었습니다" | 로그인 시간 초과 | 다시 로그인 |
| "권한이 없습니다" | 해당 기능 접근 권한 없음 | 관리자에게 권한 요청 |
| "이미 처리된 건입니다" | 중복 처리 시도 | 화면 새로고침 후 확인 |
| "필수 항목을 입력하세요" | 필수 입력 항목 누락 | 빨간색 표시된 필드 입력 |
| "서버 오류가 발생했습니다" | 시스템 오류 | 잠시 후 재시도, 지속 시 관리자 문의 |
