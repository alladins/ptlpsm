# 4. 납품 확인

## 4.1 납품확인 개요

납품확인은 현장에서 실제 물품을 수령하고 확인하는 과정입니다. 현장 담당자(인수자)가 **모바일 기기**로 직접 처리합니다.

### 납품확인 프로세스

```
출하 (배송완료 상태)
       ↓  관리자: 모바일 URL 발송 (SMS/LMS)
현장 담당자 URL 접속 (로그인 불필요, 토큰 인증)
       ↓
정보·수량 확인 → (사진 촬영) → 터치 서명 저장
       ↓
납품확인 완료
```

> **두 가지 진행 모드가 있습니다.**
> - **일반(full) 모드**: 현장 담당자가 **사진 업로드 + 서명**을 모두 진행
> - **서명전용(signature) 모드**(URL `?mode=signature`): 사진은 **관리자가 미리 업로드**하고, 현장 담당자는 **서명만** 진행
>
> 화면에 *"납품 사진은 관리자가 처리했습니다. 서명만 진행해주세요."* 가 보이면 서명전용 모드입니다.

## 4.2 관리자 화면 (PC)

### 접근 경로
- **메뉴**: 납품관리 → 납품확인
- **화면 주소**: `/admin/delivery/list`

> 📷 [스크린샷: 납품확인 목록(PC) — 발주 단위 트리 + 출하별 납품확인 상태]

### 화면 구성
납품확인 목록은 **트리 구조**입니다. 발주 단위로 그룹화되고, 각 발주 하위에 출하 건과 납품확인 상태가 표시됩니다.

### 단계별 조작 — 납품확인 URL 발송
1. 납품확인이 필요한 출하 건을 선택합니다.
2. **[납품확인 URL 발송]** 을 누릅니다.
3. **수신자명**과 **휴대폰 번호**를 입력합니다.
4. 메시지 유형(**SMS / LMS**)을 선택합니다.
5. **[발송]** 을 누릅니다.

### URL 유효기간
- 발송된 URL은 **24시간** 동안 유효합니다.
- 만료된 링크로 접속하면 *"서명 링크가 만료되었습니다. 관리자에게 새 서명 링크를 요청해주세요."* 가 표시됩니다. → 관리자가 **재발송**합니다.

## 4.3 모바일 납품확인 (현장 담당자)

### 접속 방법
- 문자로 받은 URL을 터치하면 모바일 브라우저에서 납품확인 페이지가 열립니다.
- **별도 로그인 불필요**(토큰 기반 인증).
- **화면 주소**: `/m/delivery/{토큰}` (서명전용은 `?mode=signature`)

> 📷 [스크린샷: 모바일 납품확인 화면 — 배송정보 + 품목표 + 사진/서명 영역]

### 납품확인 절차

#### Step 1 — 정보 확인
화면 상단에서 **운송장번호 · 차량번호 · 배송지 · 배송예정일 · 현장소장**과 품목 표(품목·규격·수량·비고)를 확인합니다.

#### Step 2 — 사진 (일반 모드)
1. **추가 사진 업로드** 영역에서 사진을 촬영/선택합니다. **(최대 5장)**
2. 첨부된 사진은 탭하여 크게 볼 수 있고, 잘못 올린 사진은 삭제할 수 있습니다.
   - 삭제 시 확인: *"이 사진을 삭제하시겠습니까?"* / 실패 시 *"삭제 실패: …"*
- 서명전용 모드에서는 이 영역 대신 *"첨부된 사진이 없습니다. 관리자가 이후 업로드할 예정입니다."* 가 표시될 수 있습니다.

#### Step 3 — 서명
1. **현장소장 서명** 영역에 손가락으로 서명합니다.
2. **[서명 저장]** 을 누르면 자동으로 완료 처리됩니다. (*"서명 후 서명 저장 버튼을 누르시면 자동으로 완료됩니다."*)
3. 서명 저장 시 **GPS 위치**가 함께 기록됩니다. (위치 권한이 없거나 실패해도 서명은 정상 저장됩니다 — 위치는 선택사항)
4. 성공하면 *"서명이 저장되었습니다"* 화면이 표시됩니다.

### 오류 메시지 & 대응

| 메시지 | 원인 | 대응 |
|--------|------|------|
| 서명 링크가 만료되었습니다. | URL 24시간 경과 | 관리자에게 재발송 요청 |
| 서명 저장에 실패했습니다. | 네트워크/서버 오류 | 통신 확인 후 재시도 |
| 삭제 실패: … | 사진 삭제 처리 오류 | 잠시 후 재시도 |

# 5. 납품완료계

## 5.1 납품완료계 개요
납품완료계는 모든 납품이 완료된 후 **조달청에 제출하는 공식 문서**입니다. 현장소장·감리원 서명을 받아 완성합니다.

### 납품완료계 상태 흐름
```
PENDING (대기)
   ↓ 첫 출하 생성
IN_PROGRESS (납품중)
   ↓ 모든 납품확인 완료
PENDING_SIGNATURE (서명 대기)
   ↓ 현장소장 서명
PARTIAL_SIGNED (일부 서명)
   ↓ 감리원 서명
COMPLETED (완료) → 3종 PDF 자동 생성
   ↓ 조달청 제출
SUBMITTED (제출완료)
```

## 5.2 납품완료계 목록
### 접근 경로
- **메뉴**: 납품관리 → 납품완료계
- **화면 주소**: `/admin/delivery-done/list`

목록 컬럼: 납품요구번호 · 현장명 · 상태 · 납품확인 현황(예: 5/5) · 서명 현황(현장소장/감리원) · 등록일

## 5.3 납품완료계 상세 화면
### 접근 경로
- 목록에서 행 클릭 → **화면 주소**: `/admin/delivery-done/detail/{ID}`

> 📷 [스크린샷: 납품완료계 상세 — 금액 요약 + 품목 표 + 사진 관리 + 변경 이력]

### 화면 구성
- **금액 요약**: **품목총액**(품대계 = 매출 기준 금액) + **수수료**(참고 표기)
- **품목 표** 컬럼: 순번 · 품명 · 규격 · 단위 · 단가 · 발주수량 · 추가수량 · 유효수량 · 납품수량 · 잔여수량 · 완료. 추가분 품목에는 **추가** 배지가 붙습니다.
- **변경 이력**: 일시 · 유형 · 품목 · 증감 기록

### 사진 관리 (사진대지용)
납품완료계 상세에서 현장 사진을 관리하고, **사진대지 PDF에 들어갈 사진**을 지정합니다.

| 동작 | 설명 | 안내/제약 |
|------|------|-----------|
| 사진 교체 | 기존 사진을 다른 사진으로 교체 | 성공: "사진이 교체되었습니다." |
| 사진 추가 | 사진을 추가 업로드 | "사진이 추가되었습니다. 사진대지에 포함하려면 '사진 선택'에서 지정하세요." |
| 사진대지 선택 | 사진대지에 넣을 사진 지정 | **출하당 최대 2장**, **최소 1장**. 저장 성공: "사진대지 선택이 저장되었습니다." |

- 최대 초과: *"사진대지는 출하당 최대 2장까지입니다. 다른 사진의 '대지'를 먼저 해제한 후 선택해 주세요."*
- 최소 미달: *"출하당 최소 1장을 선택해야 합니다."*
- 사진을 교체/추가/재선택하면 **사진대지 PDF가 자동으로 다시 생성**됩니다.

### 재계산 (관리자 전용)
출하 이관 등으로 수량/금액이 어긋났을 때, 상세 화면의 **[재계산]** 버튼(SYSTEM_ADMIN 권한)으로 납품완료 수량·금액을 다시 맞춥니다. 확인 창 후 실행되며, 성공 시 *"재계산이 완료되었습니다."*, 실패 시 *"재계산에 실패했습니다."* 가 표시됩니다.

## 5.4 서명 수집
### 단계별 조작
1. 납품완료계 상세에서 **[서명 URL 발송]** 을 누릅니다.
2. 서명자를 선택합니다. **현장소장 → 감리원 순서**입니다.
3. 메시지 미리보기(납품요구번호·수요기관·사업명·서명 URL)를 확인하고 **[발송]** 합니다.
4. 서명 현황은 **대기(⏳) / 완료(✅)** 로 표시됩니다. URL 유효기간은 **24시간**입니다.

## 5.5 PDF 문서 생성 및 다운로드
**현장소장 + 감리원** 양측 서명이 완료되면 3종 PDF가 자동 생성됩니다.

| 문서 | 설명 |
|------|------|
| 납품확인서 | 품목별 납품 내역 확인 문서 |
| 납품완료계 | 최종 납품완료 보고서(조달청 제출용) |
| 사진대지 | 사진대지로 지정한 현장 사진 모음(출하당 1~2장) |

다운로드: 상세 화면의 **문서** 영역에서 문서별 **[다운로드]**, 또는 **[전체 다운로드]**(ZIP).

## 5.6 조달청 제출
1. 모든 서명 완료와 PDF 생성을 확인합니다.
2. **[제출 완료 처리]** 를 누르면 상태가 **SUBMITTED(제출완료)** 로 바뀝니다.

> **주의**: 제출 완료 처리 후에는 수정이 불가능합니다.
