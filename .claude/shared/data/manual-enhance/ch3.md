# 3. 출하 관리

## 3.1 출하 목록 조회

### 접근 경로
- **메뉴**: 출하관리 → 출하목록
- **화면 주소**: `/admin/shipping/list`

> 📷 [스크린샷: 출하 목록 화면 — 검색줄 + 출하 목록 테이블(합지/추가 배지 포함)]

### 화면 구성

출하 목록은 각 출하 건의 상태·수량·금액을 한 줄로 보여줍니다. 일부 행에는 다음 배지가 표시됩니다.
- **합지**: 여러 발주 품목이 한 출하로 합쳐진 건
- **추가**: 추가변경으로 품목/수량이 더해진 건

목록 컬럼(좌→우): **출하NO · 납품요구일자 · 수요기관 · 사업명 · 출하일자 · 상태 · 출고 · 출하수량 · 출하금액 · 비고**

### 출하 상태

| 상태 | 설명 | 표시 색상 |
|------|------|-----------|
| 등록 | 출하 정보 등록 완료 | 회색 |
| 운송중 | 배송 출발 | 파란색 |
| 배송완료 | 현장 도착 | 주황색 |
| 납품확인 | 납품확인 완료 | 녹색 |

### 검색 조건

| 검색 항목 | 입력 방식 | 설명 |
|-----------|-----------|------|
| 출하일자 | 시작일 ~ 종료일 (달력) | 출하일자 기준 기간 |
| 납품요구번호 | 텍스트 (`납품요구번호 입력 또는 조회`) | 연결된 발주의 납품요구번호 |
| 검색어 | 텍스트 (`수요기관, 사업명, 출하NO`) | 수요기관·사업명·출하번호 통합 검색 |
| 상태 | 드롭다운 | 등록 / 운송중 / 배송완료 / 납품확인 |

### 단계별 조작 — 조회하기
1. 출하관리 → 출하목록으로 이동합니다.
2. 출하일자 기간·납품요구번호·검색어·상태 중 필요한 조건을 입력합니다.
3. **[검색]** 또는 입력 후 **Enter** 로 조회합니다.
4. 행을 클릭하면 해당 출하의 상세/수정 화면으로 이동합니다.

### 주요 버튼/동작

| 버튼 | 동작 | 조건 |
|------|------|------|
| 검색 | 검색 조건으로 목록 조회 | 항상 |
| 엑셀 | 조회 결과를 Excel로 내려받기 | 항상 |
| 등록 | 출하 등록 화면으로 이동 | 쓰기 권한 있을 때만(없으면 "권한이 없습니다") |

### 오류 메시지 & 대응

| 메시지 | 원인 | 대응 |
|--------|------|------|
| 등록된 출하 정보가 없습니다. | 조건에 맞는 출하 없음 | 검색 조건 완화 |
| 엑셀 다운로드에 실패했습니다. | 서버/네트워크 오류 | 잠시 후 재시도 |

## 3.2 출하 등록

### 접근 경로
- 출하목록 → 상단 우측 **[등록]** 버튼
- **화면 주소**: `/admin/shipping/register`

> 📷 [스크린샷: 출하 등록 화면 — 발주 선택 + 자동 채움 기본정보 + 품목/수량 입력 표]

### 등록 절차

#### Step 1 — 발주(납품요구) 선택
1. **납품요구번호(발주번호)** 필드의 **[조회]** 로 발주를 검색·선택합니다. (`납품요구번호를 선택하세요`)
2. 발주를 선택하면 **납품요구일자·사업명·수요기관명·담당자·건설사** 등이 **자동으로 채워집니다.** (`발주 선택 시 자동으로 입력됩니다`)
   - 발주 정보를 불러오지 못하면 → *"발주 정보를 불러오는데 실패했습니다."*

#### Step 2 — 출하 품목 및 수량 입력
1. 발주 품목 중 출하할 품목을 추가합니다.
   - 이미 추가한 품목을 또 추가하면 → *"이미 추가된 품목입니다."*
2. 각 품목의 **출하 수량**을 입력합니다.
   - 0 이하 등 잘못된 값 → *"올바른 수량을 입력하세요."*
   - 잔여수량 초과 → *"출하수량은 잔여수량을 초과할 수 없습니다."* (출하수량 ≤ 발주수량 − 기출하수량)

#### Step 3 — 공급원·운송·기타 정보
1. **공급원**(제조사/OEM)을 선택합니다. **(필수)**
2. **출하일자**를 선택합니다. **(필수)**
3. 배송비, 비고 등 부가 정보를 입력합니다.

#### Step 4 — 저장
1. 입력 내용을 확인합니다.
2. **[저장]** 을 누릅니다.
   - 출하 품목이 없으면 → *"출하할 품목이 없습니다. 출하수량을 입력해주세요."*
   - 성공 → *"출하 정보가 저장되었습니다."*

### 입력 필드 설명

| 필드 | 필수 | 설명 |
|------|:----:|------|
| 납품요구번호(발주번호) | O | 출하할 발주. [조회]로 선택 |
| 납품요구일자 | 자동 | 발주 선택 시 자동 |
| 사업명 | 자동 | 발주 선택 시 자동 |
| 출하일자 | O | 출하 날짜 |
| 상태 | - | 출하 상태(기본 '등록') |
| 공급원 | O | 제조사/OEM 공급처 |
| 수요기관명 | O | 발주처(자동) |
| 기관번호 | - | 수요기관 식별번호 |
| 담당자 / 건설사 / 현장담당자 | 자동/- | 발주 기준 자동 또는 보완 입력 |
| 배송비 | - | 운송 비용(기본 0) |
| 총 출하수량 / 총 금액 | 자동 | 품목 입력에 따라 자동 계산 |

### 오류 메시지 & 대응

| 메시지 | 원인 | 대응 |
|--------|------|------|
| 올바른 수량을 입력하세요. | 수량이 0 이하/형식 오류 | 1 이상 숫자로 입력 |
| 이미 추가된 품목입니다. | 동일 품목 중복 추가 | 기존 행의 수량을 수정 |
| 출하수량은 잔여수량을 초과할 수 없습니다. | 발주 잔여수량 초과 | 잔여수량 이내로 조정 |
| 출하할 품목이 없습니다. 출하수량을 입력해주세요. | 품목/수량 미입력 상태로 저장 | 품목 추가 후 수량 입력 |
| 출하 정보 저장에 실패했습니다. | 검증 실패·서버 오류 | 필수 필드(공급원·출하일자) 확인 후 재시도 |
| 발주 정보를 불러오는데 실패했습니다. | 발주 조회 실패 | 납품요구번호 재선택, 반복 시 관리자 문의 |

## 3.3 출하 수정 및 추가변경

### 접근 경로
- 출하목록에서 해당 행 클릭 → 출하 상세/수정 화면
- **화면 주소**: `/admin/shipping/edit/{출하ID}`

### 일반 수정
출하 등록 후 **납품확인 전**까지 출하일자·공급원·운송/배송 정보·비고 등을 수정할 수 있습니다. **[저장]** 으로 반영합니다.

### 추가변경 (수량 변경)
이미 등록된 출하의 수량을 바꿔야 할 때 사용합니다.

1. 출하 상세 화면에서 **[추가변경]** 을 누릅니다.
2. 변경 유형(수량 증가/감소)과 변경 수량, **변경 사유(필수)** 를 입력합니다.
3. **[변경 적용]** 을 누릅니다.

> **제한**: 해당 출하가 **기성금 청구에 포함된 경우** 수량 변경이 불가합니다. 이때는 새 출하를 등록하세요.

### 변경 이력 조회
출하 상세 화면의 **[변경 이력]** 에서 변경 일시 · 변경 전/후 수량 · 사유 · 변경자를 확인할 수 있습니다.
