# 1. 시스템 개요

## 1.1 시스템 소개

PTLPSM(출하관리 시스템)은 공공조달 물류/출하 업무를 통합 관리하는 시스템입니다. 발주부터 납품완료, 자금 정산까지 전체 프로세스를 지원합니다.

- **관리자 웹**(PC): 발주·출하·납품·자금 등 모든 관리 업무. 주소 체계는 `/admin/...`
- **모바일 납품확인**(스마트폰): 현장 인수자/서명자가 로그인 없이 URL로 접속. 주소 체계는 `/m/delivery/{토큰}`

> 📷 [스크린샷: 로그인 화면 및 관리자 메인 대시보드 — 좌측 메뉴 + 상단 사용자/알림]

### 주요 특징

| 기능 | 설명 |
|------|------|
| **PDF 자동 추출** | 납품요구서 PDF를 업로드하면 AI/OCR로 정보 자동 추출 |
| **모바일 납품확인** | 현장에서 스마트폰으로 서명, 사진 촬영, GPS 위치 기록 |
| **다중 서명** | 현장소장 + 감리원 서명을 통한 승인 체계 |
| **자금 추적** | 선급금 → 기성금 → 잔금 → OEM 지급 전체 관리 |
| **문서 자동 생성** | 납품확인서, 납품완료계, 선급금 문서 등 자동 생성 |

## 1.2 전체 업무 흐름

### 기본 흐름도

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  발주   │ →  │  출하    │ →  │  운송    │ →  │  납품    │ →  │납품완료  │ →  │자금관리  │
│  등록   │    │  등록    │    │  배차    │    │  확인    │    │   계    │    │  정산   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼              ▼
 납품요구서       출하지시서      운송장          모바일서명      3종PDF        선급/기성
 PDF 추출        발행           발행           사진촬영        생성          잔금/OEM
```

### 데이터 관계도

```
발주(Order) ──── 1:N ────→ 출하(Shipment) ──── 1:1 ────→ 운송(Transport)
    │                          │                              │
    │                          └──────────── 1:1 ─────────────┤
    │                                                         ▼
    │                                                  납품확인(Delivery)
    │                                                         │
    │                                                         ▼
    │                                                  인수증 PDF 생성
    │
    ├──── 1:N ────→ 기성차수(Baseline) ──→ 납품확인서/사진대지 PDF
    │
    ├──── 1:1 ────→ 납품완료계(DeliveryDone) ──→ 3종 PDF
    │
    └──── 1:1 ────→ 자금(Fund)
                      ├── 선급금 (1회)
                      ├── 기성금 (N회) ←── 기성차수와 연결
                      └── 잔금 (1회)
```

### 핵심 관계 설명

| 관계 | 비율 | 의미 |
|------|------|------|
| 발주 → 출하 | **1:N** | 발주 1건에 여러 번 분할 출하 가능 |
| 출하 → 운송 | **1:1** | 출하 1건당 운송장 1건 |
| 운송 → 납품확인 | **1:1** | 운송 완료 시 납품확인 1건 |
| 발주 → 기성차수 | **1:N** | 기성금 분할 청구 가능 |
| 발주 → 자금 | **1:1** | 발주별 자금 관리 |

## 1.3 사용자 역할

권한은 역할 코드(roleCode) 기준으로 부여됩니다. **시스템관리자(SYSTEM_ADMIN)** 와 **리드파워담당자(LEADPOWER_MANAGER)** 는 전체 기능에 접근할 수 있고(전체 접근 권한), 그 외 역할은 담당 업무로 제한됩니다. 시스템관리 메뉴(시스템 설정·접근로그·방문통계 등)는 **SYSTEM_ADMIN 전용**입니다.

| 역할 | 코드 | 주요 업무 | 접근 범위 |
|------|------|----------|----------|
| 시스템 관리자 | SYSTEM_ADMIN | 사용자/권한 관리, 시스템 설정, 통계 | 전체(시스템관리 포함) |
| 리드파워 담당자 | LEADPOWER_MANAGER | 발주/출하/납품/자금 관리 | 전체(시스템관리 제외) |
| 영업 담당자 | SALES | 수주/견적/영업 관리 | 담당 업무 |
| 현장 소장 | SITE_MANAGER | 납품확인·기성 서명 | 모바일 서명 |
| 현장 감리원 | SITE_INSPECTOR | 납품확인 검수·서명 | 모바일 서명 |
| 운송 담당자 | DRIVER | 배송 상태 업데이트 | 운송 |

> 메뉴별 노출/접근 권한은 **기초정보 → 메뉴권한(`/admin/basic-info/menu-auth`)** 에서 역할별로 관리합니다. (역할 코드 명칭이 코드와 다를 수 있어 일부는 `(확인 필요)`)

## 1.4 용어 정의

### 1.4.1 핵심 엔티티(업무 단위)

| 용어 | 영문/테이블 | 설명 | 다른 표현 |
|------|------------|------|-----------|
| 발주 | Order / orders | 공공조달 계약에 따른 물품 납품 요청 단위 | 납품요구·수주·계약 |
| 출하 | Shipment / shipments | 창고에서 현장으로 물품을 보내는 단위 | 배송·출고 |
| 운송 | Transport / transports | 출하 물품의 운송 정보(차량·기사) | 배차·운송장 |
| 납품확인 | Delivery / deliveries | 현장에서 물품 인수를 확인하는 절차 | 인수·수령확인 |
| 납품사진 | DeliveryPhoto | 납품 현장 증빙 사진 | - |
| 기성차수 | Baseline / baselines | 기성금 청구를 위한 납품 완료 단위(1·2·3차…) | 기성·차수 |
| 기성차수품목 | BaselineItem | 기성 차수의 품목별 수량/금액 스냅샷 | - |
| 자금 | Fund / funds | 발주별 자금 관리 정보 | 자금관리 |
| 선급금 | AdvancePayment | 계약 체결 후 선지급 금액(통상 70%) | - |
| 기성금 | ProgressPayment | 납품확인 완료 물량에 대한 대금 | 기성청구 |
| 잔금 | BalancePayment | 납품완료 후 정산되는 나머지(=계약금액−선급금−기성금누계) | - |
| OEM지급 | OemPayment | 협력업체 지급액(=기성금×70%) | - |
| 납품완료계 | DeliveryDone / delivery_done | 전체 납품 완료 후 조달청 제출용 보고서 | 납품완료·준공 |
| 품목 | Item / items | 납품 대상 물품 정보 | 물품·제품 |
| 회사 | Company / companies | 건설사·제조사 등 협력업체 정보 | - |

### 1.4.2 자주 쓰는 용어

| 용어 | 설명 |
|------|------|
| 납품요구번호 | 조달청 발행 고유 번호(예: `R26TB0...-00`). `-00`=본계약, `-01` 이상=변경계약 |
| 본계약 / 변경계약 / 추가계약 | 최초 계약 / 기존 계약 내용 변경 / 별도 추가 계약 |
| 품대계(품목총액) | 품목 금액 합계. **매출·집계·정산의 단일 기준 금액** |
| 발주수량/출하수량/납품수량/잔여수량 | 계약 수량 / 출고한 수량 / 납품확인된 수량 / 남은 수량 |
| 선급금비율 | 계약금액 대비 선급금 비율(기본 70%) |
| 사진대지 | 납품 현장 사진을 모은 증빙 문서(출하당 1~2장) |

## 1.5 업무 프로세스 상세 흐름

### 1.5.1 발주 → 출하 → 납품확인 사이클

발주 1건에 대해 **계약 물량을 모두 채울 때까지** 출하를 반복합니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                           발주 등록                                  │
│              (납품요구번호, 품목, 수량, 금액 등록)                    │
│                    예: 품목A 100개, 품목B 50개                       │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        출하 등록 (1차)                               │
│                   품목A: 100개 중 30개 출하                          │
│                   품목B: 50개 중 20개 출하                           │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        운송 배차 (1:1)                               │
│               차량번호, 기사명, 배송 예정일 등록                      │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     모바일 납품확인 (1:1)                            │
│                  인수자 서명 + 사진촬영 + GPS                        │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                      인수증 PDF 자동 생성                            │
│                   (출하 완료, 기성금 청구 가능)                       │
└────────────────────────────┬────────────────────────────────────────┘
                             ▼
              ┌──────────────┴──────────────┐
              │     남은 물량이 있는가?      │
              └──────────────┬──────────────┘
                    예 │           │ 아니오
                       ▼           ▼
┌──────────────────────────┐  ┌────────────────────────────────────┐
│     출하 등록 (2차)       │  │       모든 물량 납품확인 완료        │
│  품목A: 남은 70개 중 40개 │  │    → 납품완료계 처리 가능 상태      │
│  품목B: 남은 30개 중 30개 │  └────────────────────────────────────┘
└──────────────────────────┘
         │
         ▼
    (반복 진행)
```

### 1.5.2 기성금 청구 프로세스

**납품확인이 완료된 출하**만 기성금 청구에 포함할 수 있습니다.

```
납품확인 완료된 출하 확인 (인수증 서명 완료분만)
        ▼
기성 차수 생성 (청구할 출하 선택 → 품목별 수량/금액 스냅샷)
        ▼
서명 URL 발송 (현장소장 + 감리원에게 LMS/SMS)
        ▼
현장소장 서명(PARTIAL_SIGNED) → 감리원 서명(SIGNATURE_COMPLETED)
        ▼
납품확인서 + 사진대지 PDF 자동 생성 (차수 상태: CONFIRMED)
        ▼
기성금 수금 확인 (입금일·입금금액 확인 후 PAID)
```

### 1.5.3 선급금 vs 기성금 vs 잔금

| 구분 | 신청 시점 | 신청 조건 | 금액 기준 | 서명 필요 |
|------|----------|----------|----------|----------|
| **선급금** | 첫 출하 전후(계약 초기) | 출하 여부 무관, **1회만** | 계약금액 × 비율(70%) | X |
| **기성금** | 납품 진행 중(여러 차수) | 납품확인 완료된 출하 | 실제 납품수량 × 단가 | O (현장소장+감리원) |
| **잔금** | 납품완료 후 | 납품완료계 완료 | 계약금액 − 선급금 − 기성금누계 | X |

**자금 흐름 예시:**
```
계약금액: 1억원
    ├─→ 선급금: 7천만원 (70%)        ←── 첫 출하 전 신청
    ├─→ 1차 기성금: 1천만원           ←── 1·2차 출하 납품확인 완료 후
    ├─→ 2차 기성금: 1천만원           ←── 3·4차 출하 납품확인 완료 후
    └─→ 잔금: 1천만원 (1억−7천−1천−1천) ←── 납품완료계 서명 완료 후
```

### 1.5.4 납품완료 시 생성 서류

| 서류 | 내용 | 용도 | 생성 시점 |
|------|------|------|----------|
| 인수증 | 출하별 납품 내역 + 인수자 서명 | 개별 납품 증빙 | 납품확인 완료 시 |
| 납품확인서 | 기성 차수별 품목 내역 + 양측 서명 | 기성금 청구 증빙 | 기성 차수 서명 완료 시 |
| 사진대지 | 현장 사진 모음(출하당 1~2장) | 납품 현장 증빙 | 기성 차수 서명 완료 시 |
| 납품완료계 | 전체 납품 요약 보고서 | 조달청 제출 | 최종 서명 완료 시 |
| 납품내역서 | 차수별 청구 내역 정리 | 정산 자료 | 필요 시 다운로드 |

### 1.5.5 상태별 가능 작업

| 단계 | 상태 | 가능한 작업 | 불가능한 작업 |
|------|------|------------|--------------|
| 출하 | 대기 | 수정, 삭제, 운송 배차 | - |
| 출하 | 진행중 | 추가변경(수량조정) | 삭제 |
| 출하 | 완료 | 조회만 | 수정, 삭제, 추가변경 |
| 기성차수 | 작성중 | 출하 선택 변경 | - |
| 기성차수 | 서명대기 | 서명 URL 재발송 | 출하 변경 |
| 기성차수 | 확정 | PDF 다운로드, 수금 확인 | 모든 수정 |
| 선급금 | 신청 | 취소 | 재신청 |
| 선급금 | 승인/수금완료 | 조회만 | 모든 수정 |

> **중요**: 기성금 청구에 포함된 출하는 수량 변경이 불가능합니다. 수량 조정이 필요하면 새 출하를 등록해야 합니다.
