# B급 조정 (B-Grade Adjustment) Feature Implementation

**Date:** 2026-01-30
**Feature:** B급 제품 가격 조정 기능 추가 (출하 수정 페이지)

## 요구사항

1. 인수증 완료된 출하에서만 B급 조정 가능
2. 팝업에서 품목별 B급 수량/가격 저장
3. B급 품목 원가에서 차감

## 구현 내용

### 1. Frontend - 출하 수정 페이지

**파일:** `pages/admin/shipping/edit/[id].vue`

#### 추가된 기능:
- **B급 조정 버튼**: PageHeader에 "B급 조정" 버튼 추가
  - 위치: "발주서" 버튼과 "삭제" 버튼 사이
  - 표시 조건: `deliveryDoneStatus === 'COMPLETED'` (인수증 완료된 출하만)
  - 스타일: `btn-action btn-warning` (주황색 경고 스타일)

#### 추가된 코드:

**Import:**
```typescript
import BgradeItemsModal from '~/components/delivery-done/BgradeItemsModal.vue'
```

**State:**
```typescript
const showBgradeModal = ref(false)
```

**Computed Properties:**
```typescript
// B급 조정 버튼 표시 여부
const canShowBgradeButton = computed(() => {
  return shipmentData.value?.deliveryDoneStatus === 'COMPLETED'
})

// B급 모달에 전달할 deliveryDoneId
const deliveryDoneIdForBgrade = computed(() => {
  if (shipmentData.value?.deliveryDoneId) {
    return shipmentData.value.deliveryDoneId
  }
  if (shipmentData.value?.bgradeItems && shipmentData.value.bgradeItems.length > 0) {
    return shipmentData.value.bgradeItems[0].deliveryDoneId
  }
  return 0
})

// B급 모달에 전달할 출하 정보
const shipmentsForBgradeModal = computed(() => {
  if (!shipmentData.value) return []
  return [{
    shipmentId: shipmentData.value.shipmentId,
    shipmentNo: shipmentData.value.trackingNumber,
    shipmentDate: shipmentData.value.shipmentDate,
    status: 'COMPLETED',
    deliveryDoneId: deliveryDoneIdForBgrade.value,
    items: []
  }]
})
```

**Methods:**
```typescript
// B급 조정 모달 열기
const openBgradeModal = () => {
  if (!canShowBgradeButton.value) {
    alert('인수증이 완료된 출하만 B급 조정이 가능합니다.')
    return
  }
  showBgradeModal.value = true
}

// B급 조정 모달 닫기
const closeBgradeModal = () => {
  showBgradeModal.value = false
}

// B급 조정 완료 후 데이터 새로고침
const handleBgradeUpdated = async () => {
  await refreshData()
}
```

**Template:**
```vue
<!-- B급 조정 버튼 (인수증 완료된 경우만 표시) -->
<button
  v-if="canShowBgradeButton"
  class="btn-action btn-warning"
  @click="openBgradeModal"
  title="B급 제품 가격 조정"
>
  <i class="fas fa-tags"></i>
  B급 조정
</button>

<!-- B급 조정 모달 -->
<BgradeItemsModal
  v-if="shipmentData"
  :is-open="showBgradeModal"
  :delivery-done-id="deliveryDoneIdForBgrade"
  :shipments="shipmentsForBgradeModal"
  @close="closeBgradeModal"
  @updated="handleBgradeUpdated"
/>
```

## 사용 흐름

1. **출하 수정 페이지 접속** (`/admin/shipping/edit/{shipmentId}`)
2. **인수증 완료 확인**: `deliveryDoneStatus === 'COMPLETED'`이면 "B급 조정" 버튼 표시
3. **B급 조정 버튼 클릭**: BgradeItemsModal 팝업 열림
4. **출하 선택**: 완료된 출하 목록에서 선택
5. **품목 선택 & 입력**:
   - 품목 클릭하여 확장
   - B급 수량 입력 (짝수만)
   - B급 단가 입력
   - 사유 입력 (선택)
   - "추가" 버튼 클릭
6. **저장**: 백엔드 API 호출하여 B급 품목 등록
7. **원가 차감**: 백엔드에서 자동으로 OEM 원가 재계산
8. **데이터 새로고침**: 출하 상세 정보 다시 로드

## 비즈니스 규칙

1. **권한**: 인수증이 완료된 출하만 B급 조정 가능
2. **수량 제약**: B급 수량은 짝수만 입력 가능
3. **원가 차감**: B급 품목 등록/수정/삭제 시 `fund_management.oem_expected_total` 자동 재계산
4. **인수증 제외**: B급 품목은 인수증 PDF에 포함되지 않음
5. **OEM 원가 필수**: OEM 원가가 설정되지 않은 품목은 B급 조정 불가
